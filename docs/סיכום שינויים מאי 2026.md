# סיכום שינויים — מערכת אדון לוקר תקלות

**תקופה:** 25-27 במאי 2026
**גרסה לפני:** מערכת מקומית בלבד, ללא חיבור ל‑Adon Locker DB, ללא לוגין, ללא ענן
**גרסה אחרי:** מערכת פרודקשן חיה עם 4 דפי שירות ואלגוריתם תעדוף משוקלל

**URL לאחר העבודה:** https://locker-faults.srv1572400.hstgr.cloud

**סטטיסטיקה כללית:** 18 פושים ל‑main, ~1,200 שורות קוד חדשות

## 1. תשתית ופריסה

### חיבור ל‑DB של נתנאל (Adon Locker)
- חיבור ל‑Supabase של נתנאל במצב Read‑Only עם משתמש `readonly_user`
- העשרת כל endpoint שמחזיר תקלות בפרטי תלמיד (שם, ת.ז, בית ספר, טלפון הורה) באופן חי
- העשרת לוקרים בפרטים: ארון, תא, סוג נעילה, קוד תלמיד, קוד מאסטר, מספר מנעול
- Cache של 60 שניות בזיכרון כדי לא להעמיס על Supabase

**נמצאו במערכת של נתנאל:** 14,865 תלמידים, 15,188 לוקרים, 49 בתי ספר, 1,538 ארונות, 22,756 מנעולים.

### פריסה לענן — OpenClaw VPS
- בניית `Dockerfile` עם Python 3.11 + gunicorn (2 workers, פורט 5000)
- בניית `docker-compose.yml` עם labels של Traefik ל‑HTTPS אוטומטי (Let's Encrypt)
- מיקום בשרת: `/docker/locker-faults/`
- Volume לקובץ ה‑SQLite (`faults_system.db`) כך שלא נמחק בעת בנייה מחודשת
- הרצה מתחת ל‑Traefik הקיים בשרת — באותו דפוס של פרויקטים אחרים (VRPTW)

### כניסה מאובטחת — Google OAuth
- חיבור Authlib עם OAuth 2.0 מול Google
- רשימת מיילים מורשים נשלפת מקובץ JSON (`data/allowed_emails.json`) — לא מ‑ENV
- ALLOWED_EMAILS פעילים: טליה, יובל (yuvalebel7), Shilo, adonlocker
- הגדרת test users ב‑Google Cloud Console + redirect URI לפרודקשן

### פתרון בעיית IPv6
- חיבור Supabase ישיר נכשל מתוך Docker (כי השרת לא תומך IPv6)
- פתרון: מעבר ל‑Supabase Session Pooler (IPv4) עם פורמט משתמש מיוחד `<role>.<project-ref>`
- ParseFix: הוספת `ProxyFix` ל‑Flask כדי שיכיר ב‑HTTPS דרך Traefik (אחרת OAuth redirect URL נבנה כ‑HTTP ולא תואם לרישום בגוגל)

## 2. פיצ'רים חדשים

### מסך דוחות וניתוח — `/reports`
- 4 כרטיסי KPI: סה"כ תקלות, פתוחות, סגורות, דחופות פתוחות
- תרשים עוגה — התפלגות לפי **סוג תקלה**
- תרשים עמודות — התפלגות לפי **חומרה**
- תרשים עמודות — תקלות לפי **חודש** (12 חודשים אחרונים)
- תרשים עמודות — תקלות לפי **בית ספר**
- טבלת **טכנאים** עם תקלות פתוחות של כל אחד
- **סינון לפי בית ספר** עם dropdown — הכל מתעדכן מיידית
- **כפתור הדפסה** עם CSS מותאם (מסתיר ניווט וכפתורים)
- **ייצוא לאקסל** עם כל התקלות + עמודות חדשות: **שם תלמיד, בית ספר, סוג מנעול** (משולפים חי מ‑Supabase של נתנאל)

### מסך טכנאי — `/technician`
- ממשק מותאם לסלולרי (כרטיסים, לא טבלה)
- **שלב 1:** רשימת בתי ספר עם כמות התקלות בכל אחד
- **שלב 2:** לחיצה על בית ספר → רשימת **מתחמים (ארונות)** באותו בית ספר, כל אחד עם מונה של כמות תקלות פעילות, ממוין לפי כמות יורדת. מתחמים בלי תקלות פעילות לא מוצגים. אם יש תקלות דחופות במתחם מופיע חיווי "X דחופות" באדום.
- **שלב 3:** לחיצה על מתחם → תקלות של אותו מתחם בלבד, ממוינות לפי מספר תא
- **חזרה מדורגת:** מתוך רשימת תקלות חוזרים לרשימת המתחמים של בית הספר; מתוך רשימת המתחמים חוזרים לרשימת בתי הספר
- כל כרטיס תקלה מציג: שם תלמיד, ת.ז, ארון, תא, סוג מנעול, **קוד מאסטר/תלמיד מודגש**, תיאור התקלה, דגלי דחיפות/חוזרת
- **כפתורי פעולה גדולים:** סגור תקלה / WhatsApp להורה / טלפון
- **שורת סינון לפי מספר טכנאי** (1-10) — הבחירה נשמרת ב‑localStorage
- חיפוש לפי שם או ת.ז
- רענון אוטומטי כל דקה
- כפתור "חזרה למערכת" בכותרת

### שליחת WhatsApp בסגירת תקלה
- בעת לחיצה על "סגור תקלה" במסך הטכנאי:
  1. התקלה נסגרת במערכת
  2. נשלף **קוד עדכני** של הלוקר מ‑Adon Locker (לא הישן מהפעם שהתקלה נפתחה)
  3. נפתח WhatsApp עם הודעה מוכנה למספר ההורה
- **תוכן ההודעה:** "הלוקר של [שם] טופל ואפשר להשתמש בו שוב" + פרטי בית ספר, ארון, תא, קוד
- הטכנאי לוחץ "שלח" — ההודעה נשלחת ממספר הטלפון שלו
- אם להורה אין מספר ב‑DB — התקלה פשוט נסגרת בלי WhatsApp

### ממשק ניהול מיילים — `/admin`
- ממשק להוספה והסרה של מיילים מורשים בלי SSH
- ניהול קובץ `data/allowed_emails.json` — שיתוף בין כל ה‑workers
- הגנות: אי אפשר להסיר מנהל / את עצמך
- תזכורת בולטת לגבי הצורך להוסיף גם ב‑Google Cloud Test Users (שני שערים)
- הגישה מוגבלת לאדמינים (`ADMIN_EMAILS` env var)

### סינון, חיפוש, ופעולות בכמות — בטאב "ניהול תקלות"
- **חיפוש מורחב** — לפי שם תלמיד או ת.ז (לא רק שם כמו לפני)
- **מסננים חדשים:**
  - בית ספר (dropdown דינמי)
  - סוג תקלה (dropdown דינמי)
  - חומרה (1-5)
  - טווח תאריכים (מ‑ עד)
- **כפתור "נקה את כל המסננים"**
- **צ'קבוקסים בכל שורה** + "בחר הכל" בכותרת
- **סרגל פעולות שצף כשמסומנות שורות:** "סגור תקלות שנבחרו" / "פתח מחדש" / "ביטול"
- אישור לפני פעולה ("האם לסגור 5 תקלות?")
- **ספירות הסטטיסטיקה מתעדכנות לפי הסינון** — בחירת בית ספר תעדכן את "סה"כ", "פתוחות", "סגורות" וכו'

## 3. שיפורים באלגוריתם תעדוף הטכנאים

### יישור עם המתודולוגיה האקדמית (קבוצה 41)
- מיפוי 4 רכיבי הציון לפי הספרות:
  - **N (35%)** — Batching, כמות תקלות בבית ספר (Benjaafar)
  - **U (25%)** — RCM, חומרה תפקודית (WBDG)
  - **T (25%)** — Max-Min Fairness, זמן המתנה (Ghaderi)
  - **R (15%)** — Service Recovery, תקלות חוזרות (Matos)
- **TOC override** (Theory of Constraints): תקלה עם `books_stuck=True` או `is_urgent=True` עוקפת את התור — ציון 10000
- הרחבת גרדיאנט T מ‑5 ימים ל‑12 ימים (גידול עדין במשך כל שבועיים)
- שמות טכנאים בעברית "טכנאי N" — תואם לפורמט שכבר היה ב‑DB

### שמירת חלוקה ב‑DB
- לפני: `/api/schedule` רק הציג חלוקה — לא שמר אותה
- אחרי: לחיצה על "תזמן" שומרת `assigned_technician = "טכנאי N"` בכל תקלה
- כך מסך הטכנאי יכול לסנן לפי החלוקה שטליה עשתה

## 4. תיקוני באגים

### NaN שובר את הפרונט
- **בעיה:** Python שלח `NaN` ב‑JSON (לא תקני), הדפדפן קרס ב‑JSON.parse → "שגיאה בטעינת תלמידים"
- **פתרון:** המרת NaN ל‑null לפני שליחה (פנדס to_json → loads)

### OAuth redirect_uri_mismatch
- **בעיה:** Flask מאחורי Traefik בנה callback URL עם `http://...` (לא https) → גוגל דחתה
- **פתרון:** הוספת ProxyFix שלימדה את Flask להכיר ב‑X-Forwarded-Proto

### לחיצה על כרטיס בית ספר במסך טכנאי
- **בעיה:** שמות עברית עם גרשיים (כמו "מקיף ז' רביבים") שברו את ה‑HTML attribute של onclick
- **פתרון:** מעבר ל‑event delegation עם data-attribute

### Tech naming inconsistency
- **בעיה:** האלגוריתם כתב "Technician 1" באנגלית, אבל ה‑Dispatcher הקיים כתב "טכנאי 1" בעברית — סינון לא עבד
- **פתרון:** שמות בעברית מהאלגוריתם + regex בפרונט שמזהה את שתי הצורות

### Force push לדריסת הקוד הישן של יובל
- בעת תחילת המעבר ל‑Supabase, יובל כבר היה דחף קוד חדש לאחר העבודה הישנה
- אחרי שיחה ואישור — דחיפה כפויה דרסה את הקוד הישן שלו (אישר שאפשר)
- היום הקוד החדש הוא הקוד הראשי
